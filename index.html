1. Explain javascript array methods: push, pop, shift and unshift. Show how to use javascript array as queue and stack respectively.

  • Pop
    1. Syntax: array.pop()
    2. Removes (pops) the last element of an array.
    3. Changes the original array & returns the removed element.
  • Push
    1. Syntax: array.push(item1, item2, ..., itemX)
      • item1, item2, ..., itemX – items to add to array (min: 1 item)
    2. The push() method adds new items to the end of an array.
    3. Changes the length of the array and returns the new length of the array
  • Shift
    1. Syntax: array.shift()
    2. Removes the first item of an array.
    3. Changes the original array.
    4. Returns the shifted/removed element.
  • Unshift
    1. Syntax: array.unshift()
    2. Adds new elements to the beginning of an array.
    3. Overwrites the original array.
    4. Returns the new length of the array

        • async: true (asynchronous) or false (synchronous)
        • user: optional user name
        • psw: optional password

6. Explain npm, npx, nvm, package.json and package-lock.json files

  • Npm (Node package manager https://docs.npmjs.com/about-npm)
    1. Shares and borrow packages, and many organizations use npm to manage private development as well
    2. Consists of three distinct components:
      • The website
        • To discover packages, set up profiles, and manage other aspects of your npm experience
      • The Command Line Interface (CLI)
        • runs from a terminal, and is how most developers interact with npm.
      • The registry
        • a large public database of JavaScript software and the meta-information surrounding it
    3. Use to:
      • Adapt packages of code for your apps, or incorporate packages as they are.
      • Download standalone tools you can use right away.
      • Run packages without downloading using npx.
      • Share code with any npm user, anywhere.
      • Restrict code to specific developers.
      • Create organizations to coordinate package maintenance, coding, and developers.
      • Form virtual teams by using organizations.
      • Manage multiple versions of code and code dependencies.
      • Update applications easily when underlying code is updated.
      • Discover multiple ways to solve the same puzzle.
      • Find other developers who are working on similar problems and projects.
  • Npx (Node Package Execute https://www.npmjs.com/package/npx)
    1. Synopsis:
      • npx [options] <command>[@version] [command-arg]...
      • npx [options] [-p|--package <pkg>]... <command> [command-arg]...
      • npx [options] -c '<command-string>'
      • npx --shell-auto-fallback [shell]
    2. Execute npm package binaries (it’s a npm package runner)
    3. Executes <command> either from a local node_modules/.bin, or from a central cache, installing any packages needed in order for <command> to run.
    4. By default, npx will check whether <command> exists in $PATH, or in the local project binaries, and execute that. If <command> is not found, it will be installed prior to execution.
  • Nvm
    1. version manager for node.js, designed to be installed per-user, and invoked per-shell
    2. allows developers to easily switch between different versions of Node.js on their system since some projects wont work on all versions
  • package.json 
    1. Every npm package and Node.js project has a package.json file with metadata for a project. The file resides in the root directory of every Node.js package
    2. Its a configuration file for NPM that descriptive and functional metadata about a project, including its name, version, dependencies, and other information
  • package-lock.json
    1. automatically generated for any operations where npm modifies either the node_modules tree, or package.json. It describes the exact tree that was generated, such that subsequent installs are able to generate identical trees, regardless of intermediate dependency updates.
    2. This file is intended to be committed into source repositories

  • Collection
    1. collection is a group of documents

    1. In mongodb the data records are stored as BSON documents. Here, BSON stands for binary representation of JSON documents,
    2. The document is created using field-value pairs or key-value pairs and the value of the field can be of any BSON type.

    1. a type of database management system (DBMS) that stores data in a row-based table structure which connects related data elements

    1. relational database management system

SQL is a language programmers use to create, modify and extract data from the relational database, as well as control user access to the database.

    1. a programming language used to manage data stored in relational databases, which store structured data in tables.

    1. The document database designed to make it easy for developers to work with data in any form, and from any programming language.

14. Asynchronous await vs. synchronous, blocking calls vs. noblocking calls

  • Asynchronous await vs. synchronous
    1. Asynchronous await is a feature that allows developers to write asynchronous code in a more synchronous style 
    2. Synchronous call would make your entire application or thread wait until the code finished executing to continue on with anything else
  • blocking calls vs. noblocking calls
    1. Blocking call program waits for a particular task to complete before continuing with the next one. Means that if a task takes a long time to complete, the program will be "blocked" or "stuck" until the task finishes
    2. A non-blocking call, a program initiates a task and continues executing without waiting for the task to complete. Means program can perform other tasks while waiting for the first task to complete.

OAuth, short for "Open Authorization", is an open standard for access delegation, commonly used as a way for internet users to grant websites or applications access to their information on other websites but without giving them the passwords. This mechanism is used by companies to permit the users to share information about their accounts with third-party applications or websites. What is the major difference between OAuth1 and Oauth2?

  • What is the major difference between OAuth1 and Oauth2?
    1. OAuth 2.0 is a complete rewrite of OAuth 1.0
      • OAuth 1.0 was largely based on two existing proprietary protocols: Flickr’s authorization API and Google’s AuthSub.
      • Where OAuth 2.0 defines four roles, (client, authorization server, resource server, and resource owner,) OAuth 1 uses a different set of terms for these roles.
      • The OAuth 2.0 “client” is known as the “consumer,” the “resource owner” is known simply as the “user,” and the “resource server” is known as the “service provider”. OAuth 1 also does not explicitly separate the roles of resource server and authorization server.
